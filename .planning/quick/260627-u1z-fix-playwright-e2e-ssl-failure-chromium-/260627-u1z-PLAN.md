<objective>
Fix the Playwright E2E SSL failure so all 4 specs run green under docker compose.

Root cause (already diagnosed, do NOT re-investigate): the `baseURL` is `http://app:8000`, where `app` is a single-label docker-compose service hostname (no dot). Chromium's HttpsUpgrades feature auto-upgrades single-label-hostname navigations from http to https. The app serves PLAIN HTTP only, so the upgraded https request fails with `ERR_SSL_PROTOCOL_ERROR` at the very first `page.goto('/')`. Proof the server is fine: `curl http://app:8000/api/health` from inside the playwright container returns HTTP 200 (documented in 04-VERIFICATION.md, Behavioral Spot-Checks). `ignoreHTTPSErrors` does NOT help — it only suppresses TLS cert errors, not the protocol upgrade.

Purpose: Turn TEST-05..08 from PRESENT_BEHAVIOR_UNVERIFIED into VERIFIED — the runtime green-phase gate that Phase 4 verification could not reproduce.
Output: A one-line `launchOptions` addition to the chromium project in `test/playwright.config.ts`. This is a test-harness fix only — NO application code, NO docker-compose change, NO spec change.
</objective>
