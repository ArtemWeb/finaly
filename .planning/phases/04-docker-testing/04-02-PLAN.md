<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host shell -> docker daemon | The launch scripts construct the docker run command; a leaked key here would land in process args/inspect |
| .env file -> container env | OPENROUTER_API_KEY crosses here at runtime; must arrive via --env-file, never as a literal arg |
| named volume -> stop script | The stop script decides whether the DB survives |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-config | Information Disclosure | docker run invocation / .env.example | high | mitigate | Scripts use --env-file .env (no literal key arg, never visible in docker inspect); .env.example holds only placeholders; .env is gitignored (acceptance: git check-ignore) |
| T-04-05 | Denial of Service | stop scripts / named volume | medium | mitigate | stop_mac.sh and stop_windows.ps1 must not contain volume rm; start scripts omit --rm so start->stop->start preserves data (acceptance criteria enforce both) |
| T-04-06 | Tampering | idempotent re-run | low | mitigate | docker container inspect guard prevents double docker run / port double-bind; re-running start is safe |
</threat_model>
