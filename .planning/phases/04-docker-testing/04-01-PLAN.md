<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host build context -> image layers | Files in the build context can leak into image layers (secrets, caches) |
| host docker daemon -> named volume | The SQLite DB is the only durable state; volume lifecycle controls data survival |
| container -> exposed port 8000 | The only externally reachable surface; serves API + static files |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Information Disclosure | Dockerfile / build context | high | mitigate | .dockerignore excludes .env (the file itself); Dockerfile sets no ARG/ENV secret value; OPENROUTER_API_KEY only ever arrives at runtime via --env-file (verified by acceptance criteria) |
| T-04-02 | Denial of Service | SQLite named volume | medium | mitigate | Persistence relies on docker stop (SIGTERM -> lifespan shutdown) not kill -9 (Pitfall 2); verify_persistence.sh uses stop/start and never volume-rm before asserting; documented as graceful-shutdown requirement |
| T-04-03 | Elevation of Privilege | runtime container user | low | accept | python:3.12-slim default user is root; ASVS-1 treats non-root USER as best-practice not required for level 1; single-host single-user demo; documented as accepted risk |
| T-04-04 | Tampering | base images (node:20-slim, python:3.12-slim, ghcr.io/astral-sh/uv) | low | accept | Official registry images (Docker Hub library + GHCR astral-sh); digest pinning is ASVS-1 "nice to have"; accepted for a demo build |
| T-04-SC | Tampering | uv / npm installs in build | high | mitigate | No new packages added in this plan; uv sync --frozen pins to committed uv.lock; npm ci pins to committed package-lock.json; all base/registry images are official (see RESEARCH Package Legitimacy Audit — all OK, no ASSUMED/SUS/SLOP) |
