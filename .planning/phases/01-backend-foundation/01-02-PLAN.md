## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser/client → POST /api/portfolio/trade | Untrusted JSON (ticker, quantity, side) drives money-moving DB writes |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Tampering | execute_trade SQL | high | mitigate | Parameterized `?` queries only; ticker/side never interpolated into SQL |
| T-02-02 | Elevation of Privilege | trade validation bypass (negative qty, oversell, overspend) | high | mitigate | Server-side guards: quantity > 0, side whitelist, cash check on buy, shares check on sell — all raise TradeError -> HTTP 400 |
| T-02-03 | Tampering | concurrent trades double-spend cash | medium | mitigate | Read-validate-write executed inside a single connect() transaction committed atomically |
| T-02-04 | Information Disclosure | error detail leakage | low | accept | TradeError messages are domain-level (e.g. "insufficient cash"), no secrets exposed |
</threat_model>
