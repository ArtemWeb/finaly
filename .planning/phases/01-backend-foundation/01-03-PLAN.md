## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser/client → POST /api/watchlist and DELETE /api/watchlist/{ticker} | Untrusted ticker string used in DB writes and passed to MarketDataSource |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Tampering | watchlist SQL (ticker path/body param) | high | mitigate | Parameterized `?` queries only; ticker never interpolated into SQL, including the DELETE path parameter |
| T-03-02 | Tampering | invalid/garbage ticker input | medium | mitigate | Normalize to uppercase and validate non-empty alphanumeric; reject with HTTP 400 |
| T-03-03 | Denial of Service | unbounded watchlist growth via repeated adds | low | accept | Single-user simulated app; UNIQUE(user_id,ticker) prevents duplicates; no further limit needed for v1 |
</threat_model>
