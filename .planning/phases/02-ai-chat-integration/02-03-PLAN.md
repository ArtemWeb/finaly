<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> POST /api/chat | Untrusted HTTP request body (message) enters the app |
| /api/chat -> chat_service -> portfolio/watchlist | HTTP request triggers auto-executing trades/watchlist mutations |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-08 | Tampering | POST /api/chat request body | medium | mitigate | Body validated by the ChatRequest pydantic model (message: str required); empty/whitespace message rejected with HTTP 400; FastAPI returns 422 for a missing field — malformed bodies cannot reach handle_chat |
| T-02-09 | Denial of Service | /api/chat invoking a slow LLM call | medium | mitigate | The underlying litellm call already runs via asyncio.to_thread (Plan 01 T-02-03) so a slow chat request does not block the event loop / other endpoints (SSE stream, portfolio) |
| T-02-10 | Elevation of Privilege | chat endpoint auto-executes trades | high | mitigate | Endpoint delegates all execution to handle_chat -> execute_trade, which enforces the same balance/share validation as manual trades (Plan 02 T-02-04). The route adds no privileged path. |
</threat_model>
