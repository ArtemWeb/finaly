<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| backend -> OpenRouter | Outbound LLM request carrying portfolio context + user message; API key crosses here |
| LLM -> backend | Untrusted model output (free-form JSON) re-enters the app and will later drive trade execution |
| package install (uv add) | New third-party code (litellm) enters the build |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Information Disclosure | OPENROUTER_API_KEY | high | mitigate | Key read only from env (os.environ); never logged, never in responses, never written to chat_messages; .env is gitignored, only .env.example committed with a placeholder |
| T-02-02 | Tampering | LLM JSON output -> ChatResponse | high | mitigate | All model output parsed through the ChatResponse pydantic schema (model_validate_json); malformed output caught and replaced with a safe fallback ChatResponse — raw model text is never eval'd or executed as code |
| T-02-03 | Denial of Service | litellm.completion blocking the event loop | medium | mitigate | The sync litellm call runs via asyncio.to_thread so a slow/hung provider call does not block other FastAPI requests |
| T-02-SC | Tampering | uv add litellm/pydantic | high | mitigate | Both are well-known PyPI packages (litellm by BerriAI, pydantic core dependency); installed via uv against the lockfile. See Package Legitimacy note below. |

Note: This phase performs package installs (litellm, pydantic). No `## Package Legitimacy Audit` table was supplied by a researcher (no RESEARCH.md for this phase). Both packages are widely-used, non-ambiguous PyPI names explicitly mandated by the project's cerebras skill (.claude/skills/cerebras/SKILL.md) and the original design (planning/PLAN.md). They are treated as legitimate per the project's own design contract; if policy requires a blocking human-verify checkpoint for installs, the executor should pause and confirm `litellm` at https://pypi.org/project/litellm/ and `pydantic` at https://pypi.org/project/pydantic/ before `uv add`.
</threat_model>
