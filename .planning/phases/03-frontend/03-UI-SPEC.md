# Phase 3 — UI Design Contract

> Visual and interaction contract for the FinAlly trading terminal frontend. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Every locked value in this file derives from one of:
> (a) `ROADMAP.md` Phase 3 success criteria, or
> (b) `REQUIREMENTS.md` UI-01..UI-12, or
> (c) the backend API contracts already implemented in `backend/app/routes/` and `backend/app/market/stream.py`.
>
> The terminal is a single static export (`Next.js output: 'export'`), styled with Tailwind CSS, served by FastAPI. There is no shadcn preset — every visual primitive is implemented directly in Tailwind utility classes plus Recharts SVG components.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind utility classes only — no shadcn, no component registry) |
| Preset | not applicable |
| Component library | None — all primitives implemented as Tailwind-styled React components |
| Charting library | Recharts (`recharts` npm package) — used for sparklines, detailed per-ticker chart, P&L line, and `<Treemap>` heatmap |
| Icon library | lucide-react (tree-shakeable, Tailwind-friendly) |
| Font | `font-mono` Tailwind stack — `ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, "Liberation Mono", "Courier New", monospace` — terminal aesthetic |
| Styling | Tailwind CSS with `darkMode: 'class'` forced-dark (no toggle); custom palette in `tailwind.config.ts` extending `theme.extend.colors` |
| React framework | Next.js 14 with `output: 'export'` (App Router), TypeScript strict mode |

Why no shadcn: this project is a single Docker image with the frontend statically exported at build time and served by FastAPI. The locked brand palette and the "Bloomberg terminal" aesthetic call for bespoke dark surfaces, not the shadcn neutral defaults. All primitives (button, card, input, dialog) are implemented inline as Tailwind className strings — no registry, no `components.json`, no automated block pulls.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Tailwind class | Value | Usage |
|-------|----------------|-------|-------|
| xs | `p-1` / `gap-1` | 4px | Icon-to-text gaps, table cell padding, sparkline row gap |
| sm | `p-2` / `gap-2` | 8px | Compact control rows (trade bar inputs), tight stack spacing |
| md | `p-4` / `gap-4` | 16px | Default panel padding, card padding, inter-row spacing in tables |
| lg | `p-6` / `gap-6` | 24px | Section-to-section spacing inside a panel |
| xl | `p-8` / `gap-8` | 32px | Major layout gutter (between watchlist, main, chat columns) |
| 2xl | `p-12` | 48px | Page-level top/bottom margin, header vertical padding |
| 3xl | `p-16` | 64px | Reserved (not used in v1) |

Exceptions:
- Icon-only control hit areas: minimum 32px × 32px (`w-8 h-8`) regardless of icon visual size — meets WCAG 2.5.5 target size (44px is ideal but 32px acceptable for in-panel icons paired with text labels).
- Header bar height: 56px fixed (`h-14`) for visual density of the terminal aesthetic.

All values are multiples of 4 except the 56px header (14 × 4), which is still grid-aligned.

## Typography

| Role | Size (Tailwind) | Weight | Line Height | Usage |
|------|-----------------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 | 1.5 (`leading-normal`) | Default text, table cells, paragraph copy in chat |
| Label | 12px (`text-xs`) | 600 (`font-semibold`) | 1.4 (`leading-snug`) | Column headers, panel titles, ticker labels, status badges — uppercase tracked via `tracking-wider` |
| Heading | 20px (`text-xl`) | 600 (`font-semibold`) | 1.2 (`leading-tight`) | Panel headers, large numeric values (cash balance, total portfolio value) |
| Display | 28px (`text-3xl`) | 600 (`font-semibold`) | 1.1 (`leading-none`) | Header portfolio value (single most prominent number on the page) |

Numeric values throughout the terminal — prices, quantities, P&L, cash — use `font-variant-numeric: tabular-nums` (Tailwind: `tabular-nums`) so columns of digits align.

Monospace family is applied globally via `body { font-family: var(--font-mono) }` in `globals.css`.

Only two font weights are used (400 + 600). No bold (700) — terminal aesthetic stays calm.

Accent (yellow/blue/purple) reserved for:
- **Yellow `#ecad0a`**: large monetary displays (total portfolio value, cash balance), the currently-selected ticker's left border (4px solid) in the watchlist, the sparkline axis line, and ticker symbol glyphs.
- **Blue `#209dd7`**: Buy button background, SSE connected-status dot, primary CTA focus ring (`focus:ring-2 focus:ring-accent-blue`).
- **Purple `#753991`**: Sell button background, secondary CTA (e.g. "Add to watchlist" confirm), hover-state on tertiary buttons.

Accent is **never** used for: body copy, panel backgrounds, table cell text, error messages (those use `text-loss`), or anything not in the reserved-for list above.

Background split (60/30/10 by visible area):
- ~60% surface-base `#0d1117` (page canvas, gutters between panels)
- ~30% surface-panel `#1a1a2e` (panel fills)
- ~10% accent + semantic (sparklines, P&L numbers, CTAs, status indicators)

## Interaction Contract

### SSE connection (`useSse` hook)

- Endpoint: `GET /api/stream/prices` (native `EventSource`).
- Connect on `AppShell` mount; close on unmount.
- On `onopen`: set status to CONNECTED (green dot).
- On `onmessage`: parse JSON, merge into local `prices` state map keyed by ticker. Trigger flash on rows whose price changed.
- On `onerror`: set status to ERROR (red dot); the browser auto-reconnects per the server's `retry: 1000` directive. On next successful `onopen`, flip back to CONNECTED. No custom exponential backoff needed.

### Watchlist flash (UI-01)

- Each `WatchlistRow` compares incoming `price` to its previous render's `price`.
- On change: apply Tailwind class `bg-profit/20` or `bg-loss/20` (depending on `price > previous` or `<`) for 500ms, then transition back to transparent via `transition-colors duration-500`.
- Implemented with a CSS class swap driven by a `useEffect` timer; no animation library.

### Sparkline accumulation (UI-02)

- Each ticker keeps an in-memory ring buffer of the last 60 prices, appended on every SSE message for that ticker.
- Buffer lives in the `prices` map state; survives re-renders; cleared on page reload (intentional — sparkline represents since-page-load session).
- Re-renders on each append are debounced via `requestAnimationFrame` to keep the UI under 60fps even with 10 tickers streaming every 500ms.

### Ticker click (UI-03)

- Clicking a `WatchlistRow` sets `selectedTicker` in `AppShell` state.
- `MainChart` re-fetches (if needed) and re-renders with that ticker. No URL routing for ticker selection in v1 — terminal is a single dashboard.

### Trade bar (UI-07)

- Submit handler validates ticker is uppercase-alphanumeric, quantity is positive integer.
- POSTs `{ ticker, quantity, side }` to `/api/portfolio/trade`.
- On 200: dispatch a `Toast` with success copy; trigger a `refreshPortfolio()` that re-fetches `/api/portfolio` and `/api/portfolio/history`. UI updates without page reload.
- On 400: dispatch error toast using copy contract table mapping (insufficient cash / insufficient shares / no price).
- On network failure: dispatch generic network-error toast.
- Buy/Sell buttons are visually identical (full-width pair, equal weight); Buy = blue, Sell = purple per accent reservation.

### Chat panel (UI-08)

- Submit handler POSTs `{ message }` to `/api/chat`.
- While awaiting response: render a single assistant message bubble with `Thinking…` and animated dots; disable input + send button.
- On response: replace the loading bubble with the assistant's message bubble. If `actions.trades` or `actions.watchlist_changes` is non-empty, render inline confirmation chips inside the message bubble (per copywriting contract). Trigger `refreshPortfolio()` and `refreshWatchlist()` so any auto-executed trades/watchlist changes appear immediately.
- Auto-scroll message list to bottom on new message (smooth scroll).
- If `OPENROUTER_API_KEY` is not set and `LLM_MOCK` is also off, the chat panel shows the disabled empty state and POST is blocked client-side.

### Add/remove ticker (UI-10)

- Watchlist add input → POST `/api/watchlist` with `{ ticker }`. On success, append the new row and let SSE pick up the price stream.
- Remove button (×) → DELETE `/api/watchlist/{ticker}`. Remove the row from local state optimistically; revert if the request fails.

### Header live values (UI-09)

- Total portfolio value: live-computed as `cash + sum(position.qty * position.currentPrice)`. SSE pushes currentPrice updates; the header re-renders the total whenever any position's price changes.
- Cash: re-fetched via `GET /api/portfolio` on trade completion and on a 10s safety poll.
- Status dot: reflects EventSource readyState (see SSE section above).

### P&L chart (UI-05)

- On mount, fetch `/api/portfolio/history`. Render as Recharts `<LineChart>` with x = timestamp, y = total value. Y-axis formatter: `$` + compact notation (`$10.2k`). Tooltip on hover shows full value + time.
- Re-render on every `refreshPortfolio()` call (so the latest post-trade snapshot appears immediately).
- X-axis shows time of day in `HH:MM` format.

## Accessibility

- All interactive elements have visible focus rings: `focus:outline-none focus:ring-2 focus:ring-accent-blue focus:ring-offset-2 focus:ring-offset-surface-base`.
- Icon-only buttons have `aria-label` (see copywriting table for "remove" wording).
- Chat message list uses `role="log"` with `aria-live="polite"` so screen readers announce new assistant responses.
- SSE status dot has `role="status"` and `aria-label` reflecting current state.
- All numeric values use `<span>` with no special role (default text); screen readers read the digits naturally.
- Color is never the sole signal: P&L numbers also carry a leading `+` or `−` character; profit/loss cells also have text content not just a background color.
- Minimum hit target 32px × 32px for all clickable controls (per spacing exception).

## API Contract Reference (for the executor)

The frontend consumes these endpoints exactly as implemented in `backend/app/routes/`:

| Method | Path | Response shape | UI consumer |
|--------|------|----------------|-------------|
| GET | `/api/health` | `{ status: "ok", ... }` | (used by docker/health checks, not the UI directly) |
| GET | `/api/watchlist` | `[{ ticker, added_at, price, previous_price, timestamp, change, change_percent, direction }, ...]` | `WatchlistPanel` (initial load) |
| POST | `/api/watchlist` | `{ status: "ok", ticker: "..." }` | Add-ticker input |
| DELETE | `/api/watchlist/{ticker}` | `{ status: "ok", ticker: "..." }` | Remove button |
| GET | `/api/portfolio` | `{ cash_balance, total_value, positions: [{ ticker, quantity, avg_cost, current_price, market_value, unrealized_pnl, change_percent }] }` | Header, PositionsTable, PortfolioHeatmap |
| POST | `/api/portfolio/trade` | `{ ticker, side, quantity, price, cash_balance }` (200) or `{ detail: "..." }` (400) | TradeBar |
| GET | `/api/portfolio/history` | `[{ recorded_at, total_value }, ...]` ascending time | PnLChart |
| POST | `/api/chat` | `{ message, actions: { trades: [...], watchlist_changes: [...] } }` | ChatPanel |
| GET | `/api/stream/prices` (SSE) | `data: { AAPL: {...PriceUpdate.to_dict()}, GOOGL: {...}, ... }` per tick | Header total value, WatchlistRow prices/flashes/sparklines, PositionsTable current price column |

SSE event payload is the union of all currently-tracked tickers' `PriceUpdate.to_dict()`. Each ticker's `direction` field is `"up" | "down" | "flat"` and is the source of truth for flash color and sparkline stroke color.
