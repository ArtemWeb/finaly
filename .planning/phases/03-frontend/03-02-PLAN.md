## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSE stream → client state | untrusted JSON payload parsed into prices map / ring buffers |
| user input (ticker symbol) → DOM render | ticker strings rendered in rows, chips, labels |
| user input (add ticker) → backend POST | add-ticker value sent to /api/watchlist |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-04 | Tampering (XSS) | WatchlistRow / ticker rendering | high | mitigate | All tickers rendered via `{ticker}` JSX (React auto-escapes); NO `dangerouslySetInnerHTML` anywhere (V5). |
| T-03-05 | Input Validation | WatchlistPanel add-ticker | medium | mitigate | Client validates uppercase-alphanumeric as UX; backend re-validates (`watchlist.py` isalnum) as the security boundary — defense in depth (V5). |
| T-03-06 | Denial of Service | SSE flood / re-render storm | medium | mitigate | rAF-debounced state updates coalesce ~500ms ticks to one render/frame; SSE cadence is server-driven (client cannot increase). |
| T-03-07 | Information Disclosure | SSE JSON.parse failure | low | accept | Parse errors are caught and logged to console; no sensitive data in the price feed; UI keeps last good state. |
| T-03-SC | Tampering | npm installs (recharts/lucide-react usage) | high | mitigate | Packages already audited + pinned in 03-01 (RESEARCH Package Legitimacy Audit all-clear); no new package installs in this plan. |
